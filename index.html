GAMS - Group Autonomy for Mobile Systems

Project Description

Version: 1.1.1

The Group Autonomy for Mobile Systems (GAMS) project at Carnegie Mellon University is intended to provide a distributed operating environment for control of unmanned autonomous systems (UAS). The repository is composed of C++, Java, MADARA, and some LUA bindings that enable a single person to control and understand information from a swarm of devices, robots, or UAS.

GAMS is an extension of an earlier project called SMASH.

Features

Legend: Supported | Unsupported but planned

GAMS provides several key capabilities to AI developers. We highlight some of the major features below.

  • MAPE-based controller
    • Extensible algorithms (analyzing, planning, and executing)
    • Extensible platforms (sensing, analyzing)
    • Distributed auctions (minimum-bid, maximum-bid)
    • Extensible logging to terminal, file, and system log
    • Seamless integration with MADARA knowledge base
    • Checkpointing of state/knowledge
    • Seamless integration with high performance data transport
    • Predictable execution control
    • Predictable update consistency (from local agent and external agents)
  • Languages
    • C++ (full)
    • Java (should compile but does not support Pose system yet, only useful for status variable interactions)
    • Python
  • Platforms
    • Intel (x86, x64) (full)
    • Arm 3+ (full)
    • Android (full)
